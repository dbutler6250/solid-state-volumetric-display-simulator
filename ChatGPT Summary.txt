# Volumetric Display Simulator Project Summary (for Codex Agent)

## Project Goal

Build a browser-based optics simulation platform that can evolve into a volumetric display design and analysis environment.

### Version 1 Goal

Implement a **Transfer Matrix Method (TMM) Bragg Reflector Simulator** that allows users to:

* Define multilayer dielectric stacks
* Calculate reflectance and transmission spectra
* Visualize results interactively
* Explore parameter changes through a modern web UI

The simulator should be usable from both desktop and mobile browsers.

---

## Current Architecture / Tech Stack

### Chosen Stack

```text
TypeScript
React
Vite
Plotly
Three.js (future use)
GitHub Pages
```

### Rationale

* TypeScript for maintainable scientific code.
* React for scalable UI architecture.
* Vite for fast development/build workflow.
* Plotly for engineering/scientific charting.
* Three.js reserved for future visualization features.
* GitHub Pages for free static hosting.

### Hosting Model

```text
Browser
│
├── React UI
├── Simulation Engine (TypeScript)
├── Plotly Visualizations
└── GitHub Pages Hosting
```

No backend server is planned for Version 1.

All computation runs client-side in the browser.

---

## Important Decisions Already Made

### Language Choice

Python approach was considered and intentionally abandoned.

Project will be implemented using:

```text
TypeScript + React
```

rather than:

```text
Python + Dash
```

### Three.js Decision

Three.js remains part of the long-term stack but:

* DO NOT implement it in Version 1.
* Introduce only when optical visualization becomes necessary.

### Architecture Philosophy

Future-proof the codebase without overengineering.

Design the simulation engine so that:

* Additional optical structures can be added later.
* The Bragg reflector is only the first supported structure.
* Core TMM code remains reusable for future simulations.

### Outputs

```text

Transmission spectrum

```

